# list

本项目学习并实现了一个侵入式链表模板。

## “侵入式”链表的含义

所谓侵入式，是指链表本身被包含在业务数据中。

我们很熟悉的通用链表结构一般如下

```C
struct list_node{
    void *data;
    struct list_node *next;
};
```

这种链表将业务数据和链表的结构指针分开，在追求高性能，资源受限，对延迟敏感的场景下存在诸多缺陷

- 内存：每个链表节点都得`malloc`一份额外内存
- 缓存命中率：节点内存和数据内存一般是分离的，CPU读取到节点后，需要跳转再去读取实际数据，容易Cache Miss

而侵入式链表则克服了上述缺点，此外

- 一个对象可以同时属于多个链表，只需要在业务数据里插入多个链表节点即可
- 避免`void*`转换引发的类型安全

## 宏代码生成

代码中使用了宏代码生成的技巧，业务模块可以使用`declare`宏来生成专属的代码

- `##`用在宏中，表示字符串拼接。代码中传入fprefix和sprefix，保证生成独一无二的代码
